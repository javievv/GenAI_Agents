## 添加新的GenAI智能体

在向代码库添加新的GenAI智能体时，请遵循以下附加步骤：

1. 在`all_agents_tutorials`文件夹中创建您的notebook。
2. 更新README.md文件：
   - 将您的新智能体添加到README的实现列表中。
   - 根据复杂性和类型将其放置在适当的类别中。
   - 使用以下格式作为链接：
     ```
     ### [编号]. [您的智能体名称 🏷️](https://github.com/NirDiamant/GenAI_Agents/blob/main/all_agents_tutorials/your_file_name.ipynb)
     ```
   - 将`[编号]`替换为适当的编号，`[您的智能体名称]`替换为您的智能体名称，`your_file_name.ipynb`替换为您的notebook文件的实际名称。
   - 选择一个能代表您智能体的适当表情符号。
   - 插入新智能体后，请确保更新所有后续智能体的编号以保持正确的顺序。
3. 更新README.md中的表格：
   - 在表格中为您的智能体信息添加新的一行。
   - 遵循现有的表格格式，包括以下列：
     - (编号)
     - 类别
     - 智能体名称
     - 框架
     - 主要功能
   - 使用与现有条目相同的类别表情符号和格式。
   - 将您的智能体放置在适当的类别中。
   - 增加表格中所有后续智能体的编号。

例如：
```
| #  | 类别          | 智能体名称                    | 框架         | 主要功能                                                                 |
|----|-------------------|-------------------------------|-------------------|------------------------------------------------------------------------------|
| 1  | 🌱 **初学者**   | [简单对话智能体](all_agents_tutorials/simple_conversational_agent.ipynb)   | LangChain/PydanticAI | 上下文感知对话，历史管理                              |
| 2  | 🌱 **初学者**   | [您的新智能体](all_agents_tutorials/your_new_agent.ipynb)     | LangGraph         | 功能1，功能2                                                         |
| 3  | 🌱 **初学者**   | [下一个智能体](all_agents_tutorials/next_agent.ipynb)          | LangChain         | 功能1，功能2                                                         |
```

请记住，在列表和表格中，在您新插入的实现之后的所有智能体的编号都要增加。

## Notebook结构

对于新的notebook或对现有notebook的重大添加，请遵循以下结构：

1. **标题和概述：** 清晰的标题和智能体的简要概述。

2. **详细解释：** 涵盖动机、关键组件、智能体架构和优点。

3. **可视化表示：** 包括一个图表以可视化智能体的架构。我们建议使用Mermaid语法创建这些图表。方法如下：

• 使用Mermaid的graph TD（自顶向下）语法创建一个图表。
• 如果需要，您可以使用Claude或其他AI助手来帮助您设计图表。
• 将您的Mermaid代码粘贴到[Mermaid Live Editor](https://mermaid.live/)中。
• 在Mermaid Live Editor的“Actions”选项卡中，下载图表的SVG文件。
• 将SVG文件存储在代码库的[images文件夹](https://github.com/NirDiamant/GenAI_Agents/tree/main/images)中。
• 为文件使用一个适当的、描述性的名称。
• 在您的notebook中，使用Markdown语法显示图像：
  ```markdown
  ![您的智能体名称](../images/your-agent-name.svg)
  ```

这个过程确保了我们可视化表示的一致性，并使其他人可以轻松理解和可能修改图表。

4. **所需软件包：** 在实现部分的开头，使用pip install命令包括所有必要的软件包安装。例如：
   ```python
   !pip install package1
   !pip install package2
   ```

5. **实现：** 带有清晰注释和解释的逐步Python实现。

6. **使用示例：** 用一个实际的例子演示智能体。

7. **比较：** 如果可能，与更简单的智能体进行定性和定量的比较。

8. **其他考虑因素：** 讨论局限性、潜在的改进或特定的用例。

9. **参考文献：** 如果有，包括相关的引用或资源。

## Notebook最佳实践

为确保所有notebook的一致性和可读性：

1. **代码单元格描述：** 每个代码单元格前应有一个markdown单元格，其中包含描述该单元格内容或目的的清晰、简洁的标题。

2. **清除不必要的输出：** 在提交您的notebook之前，清除所有不必要的单元格输出。这有助于减小文件大小，并避免因过时的结果而引起的混淆。

3. **一致的格式：** 在整个notebook中保持一致的格式，包括定期使用markdown标题、代码注释和正确的缩进。

## 代码质量和可读性

为确保我们的代码具有最高的质量和可读性：

1. **编写清晰的代码：** 遵循清晰、可读代码的最佳实践，为每个函数和类编写文档。
2. **使用注释：** 添加清晰简洁的注释来解释复杂的逻辑。
3. **格式化您的代码：** 在您的贡献中始终使用一致的格式。
4. **语言模型审查：** 完成代码后，可以考虑通过语言模型进行额外的格式化和可读性改进。这个额外的步骤可以帮助您的代码更易于访问和维护。
